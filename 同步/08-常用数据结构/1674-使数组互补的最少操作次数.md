# 1674. 使数组互补的最少操作次数

🔗 [题目链接](https://leetcode.cn/problems/minimum-moves-to-make-array-complementary/)   📊 难度分 **2333**   ⏱️ 耗时 **30 分钟**   📂 08-常用数据结构

> 💡 **关键词**:差分数组+区间覆盖

## 思路
对于每一对 (nums[i], nums[n-1-i])，其和可以落在三个区间：不需要修改（和为原和）、修改一次（区间 [min+1, max+limit] 除去原和）、修改两次（其余区间）。使用差分数组对每个区间进行加减操作，最后遍历所有可能的和，取最小操作次数。

## 关键点
- 将每对数的操作代价按区间分类，用差分数组高效维护
- 注意处理原和所在位置，需要从一次操作区间中扣除
- 差分数组长度设为 2*limit+2，覆盖所有可能的和

## 复杂度
- 时间:O(n+limit)
- 空间:O(limit)

## 我的代码

```python
class Solution:
    def minMoves(self, nums: List[int], limit: int) -> int:
        #差分数组
        #对某一段连续子数组[l,r]+k,可以让l+k,然后r+1-k,然后这样的话就可以进行累加还原为之前的
        n=len(nums)
        diff=[0]*(limit*2+2) #
        # 比如limit=2
        # 那就可以2，3，4
        # diff可以从2，3，4，5，对应0，0，2，1，1，-2所以需要长度为2*limit+2
        for i in range(n//2):
            x=nums[i]
            y=nums[-1-i]
            #如果只修改y，则x+y=[x+1,x+limit]
            #只修改x,x+y=[y+1,y+limit]
            #取并集，这部分修改的代价是1（不包括其本身）
            l=min(x,y)+1
            r=max(x,y)+limit
            #其余空间[2,l-1]∪[r+1,2*limit]都代价需要2

            #因此对连续子数组进行修改，可以使用差分数组

            #[2,l-1]
            diff[2]+=2
            diff[l]-=2

            #[l,r]
            diff[l]+=1
            diff[r+1]-=1

            #去掉自身
            diff[x+y]-=1
            diff[x+y+1]+=1
```
