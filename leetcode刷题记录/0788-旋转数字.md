# 788. 旋转数字

🔗 [题目链接](https://leetcode.cn/problems/rotated-digits/)   📊 难度分 **1397**

> 💡 **关键词**:数位DP+状态标记

## 思路
题目要求统计1到N之间每位旋转后仍有效且与原数不同的数字个数。由于N最大10000，但数位DP可扩展至更大范围。核心是逐位枚举数字，用`hasdiff`标记是否已包含2/5/6/9（即旋转后变化的数字），同时排除包含3/4/7的无效数字，最终统计所有有效且变化的数字。

## 关键点
- 用`tmp`数组预定义每个数字旋转后的状态：0（不变）、1（变化）、-1（无效）
- 通过`hasdiff`布尔值记录是否已出现变化数字，确保最终数字与原数不同
- 利用`islimit`控制当前位枚举上限，避免重复计算

## 复杂度
- 时间:O(len(N) * 2 * 2 * 10) = O(10 * len(N))
- 空间:O(len(N) * 2 * 2)

## 我的代码

```python


class Solution:
    def rotatedDigits(self, n: int) -> int:
        
        #数位动态规划
        s=str(n)
        slen=len(s)
        #0,1,8反转后是自己，2，5，6，9反转后为镜像，其他为-1
        tmp=(0,0,1,-1,-1,1,1,-1,0,1)
        #找i之后的位置，进行数字填充
        #子串中至少要包含一个2569，且不能包含347
        #另外进行填充的时候，要看是否是tight,
        #因为如果前面的数字都和n对应位置上的数字相等的话，那当前位最大值也就是n对应位
置最大值
        #如果前面的数字有一个小于n，那从这个数字后面开始最大值可以是9
        @cache
        def dfs(i,hasdiff,islimit):
            if i==slen:
                return hasdiff
            res=0
            up=int(s[i]) if islimit else 9
            for j in range(0,up+1):

                #j！=347
                if tmp[j]!=-1:
                    res+=dfs(i+1,hasdiff or tmp[j],islimit and j==up)
        
            return res
        return dfs(0,False,True)
```
